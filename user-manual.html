Getting Started

Platform Overview

The Ava Ocean platform is a web-based tool for recording, monitoring, and analysing data from the Ava Ray's scallop harvesting operations. It has four main parts, each designed for a different user and purpose.

📱
Captain's Log Captain
Mobile-optimised data entry at sea. Log activities, tows, bycatch, and daily catch weight in real time.
📊
Operations Dashboard DagnyØystein
Charts, tables, maps, and drill-down analysis of all operational data. CPUE, emissions, bycatch.
🔬
Research Dashboard Dagny
PhD data collection and analysis — Shannon diversity, BACI design, damage index, species records.
🏭
Factory Log Dagny
Enter factory sample data — meat yield, damage rates, population split (% female/male).
Access URL: All tools are at https://ava-ocean-platform.vercel.app — bookmark this on your phone and computer.

Getting Started

Logging In

1
Open the platform — go to ava-ocean-platform.vercel.app in your browser. On mobile, you can add it to your home screen for quick access.
2
Choose your tool — from the home page, tap Captain's Log, Dashboard, or Research Dashboard.
3
Enter your credentials — use your assigned email and password. Contact Dagny if you need access.
4
Language — toggle between Norwegian (NO) and English (EN) using the buttons at the top. Your preference is remembered.
Tip: The platform works offline for reading data you've already loaded. If your satellite internet drops at sea, data you've entered will still save once connection returns — but always check the green status dot in the header to confirm you're connected before logging critical tows.
Captain at Sea

Starting a Trip

1
Open Captain's Log and log in with your captain credentials.
2
Select an existing trip from the dropdown if one has already been created, then tap "Fortsett denne turen →".
3
Or create a new trip — tap "+ Ny tur" and fill in: trip code (e.g. AVA-2026-001), your name, departure date, fishing site, and departure port.
4
Tap "Opprett tur" to save. The trip is now live in the database and visible on the dashboard onshore.
Demo/Training mode: If you're practising or training a new crew member, toggle on "Demo / treningstur" when creating the trip. This marks all data as test data that can be deleted with one click from the Admin panel. Never use this on a real fishing trip.
Captain at Sea

Logging Activities

Every hour at sea should be accounted for as one of five activity types. This is what builds the time-use analysis and fuel efficiency data.

🦪
Fiske / Fishing
Harvester is in the water and actively collecting scallops.
⚓
Transitt / Steaming
Vessel moving between sites or to/from port.
🌊
WOW
Waiting On Weather — stopped due to sea conditions.
⚙️
Nedetid høster
Harvester equipment downtime (mechanical, maintenance).
🏭
Nedetid fabrikk
Factory/processing equipment downtime.
1
Tap the activity type you are starting (e.g. 🦪 Fiske).
2
Tap "Start aktivitet →"
3
Fill in conditions: fuel level at start (litres), wave height (m), wind speed (knots), wind direction, sea state, visibility. These fields are optional but very valuable for research — fill them in when you can.
4
GPS position: Tap "📍 Hent start-GPS" to capture your position automatically, or type coordinates manually if GPS doesn't work.
5
Tap "Start aktivitet ▶" to begin. The activity is now live.
6
When done: tap "Avslutt" in the active activity banner, enter end fuel level and GPS, then confirm.
Only one activity at a time. You must end the current activity before starting a new one. The platform will warn you if you try to start a second activity while one is open.
Captain at Sea

Logging a Tow (Høsting)

Tows are logged inside a Fishing activity. Each time the harvester completes a pass and the basket is emptied, that is one tow.

1
Start a Fishing activity (see above). You'll see the tow list screen.
2
Tap "+ Logg ny høsting" when you begin a tow.
3
Start time and GPS — the current time is pre-filled. Capture or type your start position.
4
End time and GPS — when the tow is complete, fill in the end time and position.
5
Harvester fill % — use the slider to record how full the collection basket is. This is the most important field — be as accurate as possible. 0% = empty, 100% = completely full (28,000 litres).
6
Depth (m) — record the fishing depth.
7
Tap "Lagre høsting og logg bifangst →" to save and proceed to bycatch entry.
GPS accuracy: GPS positions are used to calculate swept area and CPUE. Even approximate positions are better than nothing — if the GPS button doesn't work, type coordinates manually. Format: decimal degrees (e.g. 74.123456, 19.654321).
Deleting a tow: If you made an error, tap the 🗑️ button next to the tow in the list. This also deletes any bycatch logged for that tow. You'll be asked to confirm before anything is deleted.
Captain at Sea

Bycatch Entry

After saving each tow, you'll see the bycatch screen. Record any non-target species that came up with the catch before they are returned to the sea.

1
Tap "+ Legg til art" for each bycatch species.
2
Enter the Norwegian common name (e.g. "Torsk", "Sjøstjerne") and select the category (fisk, virvelløs, krepsdyr, etc.).
3
Count by size: estimate numbers in Small, Medium, Large groups. No need to be exact — a rough count is fine.
4
"Returnert til sjøen?" — toggle on if the species was returned alive (this should almost always be on).
5
Tap "Lagre og fullfør ✓" when done, or "Ingen bifangst — hopp over" if there was nothing.
This data is important. Bycatch records are part of the commercial research licence requirements and feed directly into Dagny's PhD research on biodiversity. Even a rough count of "5 starfish, small" is valuable.
Captain at Sea

Daily Catch Log

Once per day, log the total kg landed for that day. This is separate from the tow-by-tow fill percentage — it's the actual processed catch weight from the factory.

1
From the main screen, tap "📊 Daglig fangstlogg".
2
Today's date is pre-filled. Enter the kg landed for the day.
3
The tow count for today is shown automatically — this is calculated from the tows you've already logged.
4
Tap "Lagre daglig logg". If you log the same date twice, the second entry updates the first — no duplicates.
Captain at Sea

Closing a Trip

When the vessel returns to port, close the trip to finalise all metrics. This calculates the complete CPUE, emissions, and area swept for the voyage.

1
Make sure all activities are ended — there should be no open (orange) activities in the list.
2
Tap "🏁 Avslutt tur og legg inn fangstvekt" at the bottom of the main screen.
3
Enter the return date and total processed meat weight (kg) from the factory for the whole trip.
4
Tap "Beregn og avslutt tur ✓". The platform calculates and displays: total tows, processed catch, catch per tow, total fuel, total CO₂, emissions per kg, area swept, and CPUE.
GPS note: If some tows are missing GPS data, you'll see a warning showing how many tows couldn't be included in the area/CPUE calculation. The catch and fuel figures are still complete.
Captain at Sea

Demo / Training Mode

When training new crew or running a demo of the platform, use Demo mode to keep test data completely separate from real data.

1
When creating a new trip, scroll to the bottom of the new trip form.
2
Toggle on "Demo / treningstur" (the orange toggle). The trip will be marked with a 🧪 badge in the trip list.
3
Log data as normal — everything works identically to a real trip.
4
When done, Dagny can go to Dashboard → Admin → "Slett alle demo-turer" to wipe all test data in one click.
Important: Never toggle Demo mode on for a real fishing trip. Demo data gets deleted — you would lose all records for that trip.
Operations Dashboard

Overview & Drill-Down

The Overview tab is the starting point — four headline numbers at the top, followed by charts, and then the interactive trip drill-down below.

Stat cards
Total trips, total tows, average fill %, total bycatch
These update automatically when you select a trip in the sidebar filter. "All trips" shows the full database.
Charts
Fill % per tow, activity breakdown, fuel/time per activity
The fill chart shows the last 30 tows in chronological order. The doughnut shows time split across activity types. The bar chart shows total hours per activity type.
Drill-down
Trip → Date → Tow hierarchy
Click any trip card to expand it. You'll see a banner with trip totals, a date-by-date table, and a map of all tows coloured by fill%. Click any date row to see the detailed tow table and 17-column daily summary for that day. The map updates to highlight only that day's tows.
Trip filter: The dropdown at the bottom of the sidebar filters all tabs simultaneously. Select a specific trip to see only that trip's data across every chart and table.
Operations Dashboard

CPUE & Emissions

This tab shows performance metrics for closed trips. A trip must be closed (via Captain's Log) before it appears here.

Trip banners
CPUE, area covered, catch per tow, emissions per kg
One banner per closed trip. CPUE = catch volume (m³) per area swept (m²). Only tows with GPS data contribute to area calculations — the banner shows how many tows had GPS.
Daily summary table
17-column breakdown per day
Shows harvest time, kg/hour, kg/tow, downtime percentages, WOW hours, average gram/meat, fuel consumption — one row per operating day.
Monthly emissions table
CO₂ per month across all trips
Aggregates fuel consumption by month, calculates total CO₂ (using 3.114 kg CO₂/litre HFO), and shows kg CO₂ per kg landed — the key emissions intensity metric.
Operations Dashboard

Fangstreg Import

Import official catch data from the Fangstreg government reporting system. This ensures the daily kg figures in the platform match the official landing records.

1
Export from Fangstreg: Log in to Fangstreg → Rapporter → Halrapport → Eksporter CSV. Save the file.
2
Open Dashboard → Fangstreg Import tab.
3
Drag and drop the CSV file onto the drop zone, or click to select it.
4
Review the preview table — rows marked "+ Ny rad" are new entries; rows marked "↺ Oppdateres" will update existing entries. Fangstreg data always wins over manually entered data on the same date.
5
Select the trip from the dropdown to link the data to the correct trip, then tap "Importer til Supabase".
6
Review the result summary showing rows inserted, updated, and any errors.
File format: The importer expects Halrapport format — UTF-16 encoding, tab-separated, with a "Haneskjell" column for scallop kg. Other CSV formats will not parse correctly.

Admin Tools

Admin access: This tab is only visible to dagny@avaocean.no. All actions are permanent and cannot be undone. Always double-check before confirming any deletion.
Demo / training data
Delete all demo trips at once
Shows a count of all trips marked as demo. Tap "Slett alle demo-turer" to delete them all, including all linked activities, tows, bycatch, and research data. Use after training sessions.
Delete single trip
Remove one trip and all its data
Select a trip from the dropdown and tap "Slett valgt tur". A confirmation dialog shows the trip name. All linked data is deleted in the correct order to avoid database errors.
Delete single tow
Remove one tow from a trip
Select a trip, then select the tow from the second dropdown (shows tow number, timestamp, and fill%). Bycatch linked to that tow is also deleted.

Select a trip, then select the date from the second dropdown (shows kg landed). Only the daily_log record is deleted — tows for that day are unaffected.
Research Dashboard

Research Sessions

Research data is organised into sessions. A session represents one data collection event — either a commercial sampling session tied to a tow, or a dedicated BACI control station visit.

Commercial sampling session
Tied to a captain's tow
The captain's tow IS the transect. The platform auto-creates the transect record from the tow data. Select the trip and tow, set BACI designation (impact/control) and before/after, then proceed to sample entry.
BACI control session
Dedicated control station visit
Not tied to a commercial tow. Enter GPS coordinates, depth, and timing manually. Set baci_designation = 'control'. Extrapolation factor is set to 1 (no scaling — full sample processed).
Research Dashboard

Entering Research Data

1
Select or create a session — link to a trip and tow (commercial) or enter manually (BACI control).
2
Enter catch sample details — number of baskets total, baskets subsampled, fill percentage. The platform calculates subsample volume and extrapolation factor automatically.
3
Species records — for each species in the subsample, enter count and mean damage index. The extrapolated total is calculated and stored.
4
Scallop measurements — enter individual shell width (mm) and damage index (0/1/2) for each measured scallop. The platform flags undersized individuals (<55mm) automatically.
5
Save the session. Shannon H' is calculated automatically from the species record counts using the extrapolated totals.
Troubleshooting

Common Issues

Problem
Red status dot — "Feil" / "Error"
The platform can't reach the database. Check your internet connection. If connection is fine, the Supabase server may be briefly unavailable — wait 30 seconds and refresh. Data you've entered in the current session is not lost.
Problem
GPS button does nothing or shows error
Your browser may have denied location permissions. On your phone, go to browser settings and allow location for this site. Alternatively, type coordinates manually — the fields accept decimal degree format (e.g. 74.123456).
Problem
Tow or activity isn't showing on dashboard
The dashboard caches data on load. Click the "↻ Oppdater" button in the relevant tab, or refresh the page. If the record still doesn't appear, check in Supabase Table Editor that it was saved.
Problem
Can't start a new activity — "En aktivitet pågår allerede"
There is an open activity that hasn't been ended. Look for the orange "Avslutt" banner at the top of the main screen and end that activity first.
Problem
Fangstreg import fails — "Mangler nødvendige kolonner"
The file format is wrong. Make sure you export "Halrapport" (not another report type) from Fangstreg, and that you export as CSV (not Excel). The file must have a "Dato" and "Haneskjell" column.
Need help? Contact Dagny at dagny@avaocean.no for access issues, data corrections, or anything not covered here.
